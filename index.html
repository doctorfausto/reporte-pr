Servidores Debian en LAN1 y LAN2

Cada servidor cuenta con una instancia de nginx y bind9, así como 3 honeypots.

Reglas Outbound de Firewall

Se implementaron dos conjuntos de reglas: uno para TCP para los protocolos HTTP y HTTPS, y otro para UDP para los protocolos DNS y NTP.

Generadores de Tráfico

Utilizando hping3, logramos generar tráfico suficiente para hacer un pequeño ataque DoS.

Pruebas internas

Se utilizó hping3 para realizar los ataques DoS y DDoS, y se utilizó Zap y Ettercap para los ataques a servicios web y DNS.

Servidores Debian y Honeypots

Para crear los honeypots, nos apoyamos de una herramienta llamada "qhoneypots" que nos permitía
crear honeypots para diferentes puertos y protocolos.
Para la computadora de la izquierda creamos honeypots para los protocolos TELNET, MySQL y Redis,
y para la computadora de la derecha creamos honeypots para los protocolos SSH, HTTP y FTP.

Configuración:
honeypots --setup redis,mysql,telnet
telnet localhost # telnet honeypot
mysql -h 127.0.0.1 -u manolo -p # mysql honeypot
redis-cli -u redis://manolo:password@127.0.0.1:6379 # redis honeypot

Reglas Outbound de Firewall en DigitalOcean

Generamos dos conjuntos de reglas en los dos DOs para tener acceso limitado a Internet:

  • El primer conjunto de reglas consiste en habilitar HTTP y HTTPS mediante el protocolo TCP
  • El segundo conjunto de reglas consiste en habilitar DNS y NTP mediante el protocolo UDP

Generadores de Tráfico

Utilizando hping3, generamos una especie de ataque DoS hacía otra computadora
dentro de la misma red, notamos una ralentización de su sistema de hasta 1000 ms.

Ejemplo:
hping3 -S --flood -p 443 -d 1024 -V --faster 10.21.1.15

Pruebas internas

  • Ataque DoS: como se mencionó anteriormente, utilizamos hping3 para hacer un ataque DoS.
  • Ataque DDoS: para el ataque DDoS basta con tener múltiples instancias de hping3 en
    distintas computadoras, todas atacando a la misma computadora.
  • Ataque a servicios web: utilizando la herramienta OWASP.
  • Ataque a servicio DNS: utilizando Ettercap, se realizó un scan de las IPs que se
    encuentran en la red para posteriormente hacer ARP poisoning e infectar la tabla de
    resolución del DNS.

Ataque "Man-in-the-middle"

Para hacer el ataque de "man-in-the-middle", utilizamos igualmente Ettercap.
Utilizamos ARP Poisoning el cual consiste en modificar la tabla ARP para
hacer creer a una computadora que el "atacante" en realidad es el router.
Además, nos apoyamos de Wireshark para analizar los paquetes.
